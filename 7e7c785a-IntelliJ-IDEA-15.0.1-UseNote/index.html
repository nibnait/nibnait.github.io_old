工具配置笔记

IntelliJ IDEA 15.0.1使用笔记

File -Settings -> Editor -> Code Style -> File Encodings: Default encoding for properties files: 全都改为utf8!

#修改一些快捷键配置：
File -Settings -KeyMap

Line Numbers: Caps Lock
Generate: Alt+Shift+S
Delete Line: Ctrl+D
Reformat Code: ctrl+shift+f
Close: ctrl+w
(Extract)Variable：alt+shift+L
ctrl+shift+U –> ctrl+shift+X 改变大小写

一些常用快捷键：

  • ctrl+alt+O： 一键导包
  • 先ctrl+F,再F3：查找并替换
  • 双击shift： 查找文件

  • alt+shift+↑：将选中行 上移

  • 生成main函数： main， ctrl+j, Enter

创建mave项目

  1. 在填写完GroupId，ArtifactId和Version之后，
    User settings file、Local repository一定要选本地maven下的。
    一定要在Properties中添加一个参数
    archetypeCatalog=internal不加这个参数，在maven生成骨架的时候将会非常慢，有时候会直接卡住。
    来自网上的解释：
    >
    archetypeCatalog表示插件使用的archetype元数据，不加这个参数时默认为remote，local，即中央仓库archetype元数据，由于中央仓库的archetype太多了，所以导致很慢，指定internal来表示仅使用内部元数据。

  1. File->Project Structure

    • Project->选择恰当的Project language level

    • Modules->

      • 在main文件夹下新建一个java文件夹，把它设为源代码文件夹。
      • 在src下建个test文件夹，在test文件夹下新建一个java文件夹，把它设为源代码文件夹。

  2. 添加 Tomcat Server

    • On ‘Update’ action和On frame deactivation全部选择Update classes and resources
    • VM options不需要填，为空就好了
    • 在Deployment中 添加 ×××：war
    • 默认发布到服务器根目录上了，

安装JRebel插件实现代码热部署

  • File->Settings->Plugins->Install plugin from disk
    jr-ide-idea-6.2.0-idea.zip（点击下载）

  • 激活方法：
    JRebel->Offlin mode
    服务器激活，

    • http://idea.lanyus.com/{username}
    • your email address

  • 在Project Structure -> Modules中 添加JRebel, Enable

添加自己的jar包到Maven仓库

以已添加我的这个page-1.0-JDK1.7.jar为例：

  • 可将page-1.0-JDK1.8.jar直接放在D盘的根目录下。

  • 打开CMD，进入这D:/ 执行：

    mvn install:install-file -Dfile=page-1.0.jar -DgroupId=com.nibnait -DartifactId=common-pagination -Dversion=1.0 -Dpackaging=jar
